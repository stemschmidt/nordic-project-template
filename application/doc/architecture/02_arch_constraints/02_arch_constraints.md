# 2 - Architectural Constraints {#page_02_arch_constraints}

## Hardware Constraints

- MCU type (e.g., nRF54L05)

- Flash / RAM limits

- Available peripherals

- Button / LED count

- No display

- Power supply constraints

## Software Constraints

- Zephyr RTOS

- No dynamic allocation (if applicable)

- MISRA compliance

- No exceptions / RTTI (if C++)

## Regulatory Constraints (if applicable)

- IEC 61508 / ISO 26262 / IEC 62304

- Radio certifications

- Security requirements
